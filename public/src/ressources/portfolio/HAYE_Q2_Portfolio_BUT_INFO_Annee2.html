Un exemple de pattern que j'ai utilisé fut le pattern Singleton.
Voici un apercu technique de l'implémentation de Singleton dans une application :





L'idée de ce pattern est très simple : on souhaite garantir qu’il n’existe qu’une seule instance d’une classe dans le programme et fournir un point d’accès global à cette instance.


Dans le cas de notre SAE, ils nous a permit de n'avoir qu'une seule connexion possible faite à notre base de données à la fois, ce qui permet d'éviter une surchage de communication inutile, ainsi que de sécuriser cette dernière.

Application de Poursuite d'Etude

Nuit de L'Info 2024 - RaceForWater

Durant ce semestre, j'ai participé à la Nuit de l'info 2024.





Nous avons réalisé avec mon équipe une application ludique faite avec Node.js et le framework Express, pour la fondation Race for Water, qui a pour mission de préserver l'eau et les océans en particulier face à la pollution plastique.


Le concept de l'application est un océan navigable, similaire au fonctionnement de Google Maps, dans laquelle se trouve différents organes interactifs. Notre objectif était de sensibiliser sur l'état actuel des océans et leurs lien méconnu et particulier avec notre corps humain.


Mon rôle fut de concevoir le serveur chargé de récuperer les réquetes et de renvoyer la page web demandée. J'ai donc du concevoir cela de façon optimisé de sorte à eviter la surcharge de rêquetes lorsque l'utilisateur souhaite accéder aux différentes pages que renvoie les organes.


Le code source de l'application est disponible sur cette page.
